# Summary of Changes - April 10, 2026

All changes in the project have been organized and divided into 5 logical groups to ensure code structure and easy tracking.

## 1. Notification System Upgrade
- **Firebase Cloud Messaging (FCM)**: Fully integrated Google's push notification system to send and receive messages in real-time.
- **Notification Badges**: Added an unread notification counter on the Bottom Navigation Bar professionally.
- **Services**: Created `PushNotificationService` and `NotificationApiService` to handle the payload and API for notifications.
- **Config**: Configured Android settings (Permissions & Metadata) to handle notifications in the background.

## 2. Localization Refactoring
- **Native Flutter l10n**: Migrated from a manual localization translation system to the official Flutter l10n system using `l10n.yaml`.
- **Cleanup**: Deleted the old `TranslationService` and the `translator` library to reduce app size and improve performance.
- **Type Safety**: Translations are now accessed in a type-safe manner via `AppLocalizations`.

## 3. UI/UX & Sync Improvements
- **Real-time Dashboards**: Updated both the Doctor and Patient dashboard screens to work with real-time bidirectional synchronization when data changes.
- **Bug Fixes**: Resolved layout Overflow issues across various screens and redesigned the `StatCard` to support interaction.
- **Onboarding & Auth**: Minor improvements to the registration and onboarding experience.

## 4. Maintenance
- **Cleanup**: Deleted temporary backup files (`.recovered`).
- **Platform Files**: Updated MacOS and Windows linker files to match the newly added dependencies.

## 5. Notifications Management & Reliability
- **Heads-Up Notifications**: Enabled High-priority Banner notifications with sound and vibration to guarantee user attention, mimicking WhatsApp's professional behavior.
- **Android 14 Compatibility**: Fixed `fullScreenIntent` permission restrictions to comply with Android 14 policies.
- **Notification Formatting**: Pre-appended the sender's name explicitly to the local notification's body to ensure visibility, and fixed the issue where "New Message" appeared as a static title when receiving WebSocket real-time messages while the app was open.
- **Notification Actions**: Built the UI and logic for clearing notifications (Mark As Read, Delete) integrated with the Backend (`NotificationApiService` & `NotificationScreen`).
- **Auto-Clear Badges**: Integrated automatic badge clearing and notification state reset when opening the Patient or Doctor chat screens.
