# Changelog: April 28, 2026

## 1. Profile Management & Backend Synchronization
- **Created `ProfileScreen`:** A fully functional profile page where the user can view and edit their personal information.
- **Biometric Adjustments:** Restricted editing of physical metrics (Height and Weight) to Patients only.
- **Image Picker Support:** Implemented picking a profile picture using `image_picker` and `path_provider`, and caching it locally using `Hive` via `StorageService`.
- **Backend API Integration (`UserApiService`):** Added a new REST service to sync user changes (name, phone, height, weight) directly with the Spring Boot backend (`PUT /api/user/me`).
- **Dashboard Auto-Update:** Modified both `PatientDashboardScreen` and `DoctorDashboardScreen` to automatically reload the cached `_currentUser` upon returning to the dashboard tab from the profile tab. This ensures immediate reflection of name or profile updates.
- **Localization (`app_en.arb`, `app_ar.arb`):** Added translation strings for all the new profile fields, success messages, and validation prompts.

## 2. Advanced UI Micro-Animations & Refinements
- **`VitalCard` Enhancements:**
  - **Realistic Heartbeat:** Replaced the simple square-wave animation with an accurate, geometric PQRST triangular ECG wave.
  - **Proportional Battery Level:** The battery charging wave height is now dynamically calculated based on the actual battery percentage value.
  - **Blood Oxygen:** Implemented smooth, floating bubbles animation.
  - **Subtle Mode (Doctor View):** Added a `subtleMode` flag. If true, the background animation opacity is reduced by 65%. Applied this to `PatientDetailScreen` so doctors aren't distracted.
- **`StatCard` & `StatusCard` Enhancements:** Introduced breathing gradients, soft pulses, and sweeping micro-animations to elevate the overall "premium" feel of the UI.

## 3. Authentication & Role-Specific Validation Logic
- **`RoleSelectionScreen` & `AuthScreen`:**
  - Standardized password validation across login and registration flows to enforce a strict minimum of 8 characters.
  - **Conditional Biometrics Registration:** Hide `Date of Birth`, `Height`, and `Weight` fields during the registration process if the user selects the "Doctor" role. These fields are now exclusively reserved for patients.
- **Models & Services:** Updated `RegisterRequest` and `UserModel` to properly map the conditionally gathered fields.

## 4. Dependencies & Platform Configs
- **`pubspec.yaml`:** Added dependencies for `image_picker`, `path_provider`, and related generation lock updates.
- **Android & iOS Pluggins:** Updated `AndroidManifest.xml` (added internet/camera permissions if applicable) and CMake lists for desktop plugins as a side-effect of adding `path_provider` and `image_picker`.

## 5. Token Handling & App Lifecycle Stability
- **`DioClient` & `TokenService`:** Improved token validation caching and centralized navigation-on-unauthorized to prevent redundant login dialogs.
- **`SplashScreen`:** Optimized initial connectivity checks and token validation before routing the user.
