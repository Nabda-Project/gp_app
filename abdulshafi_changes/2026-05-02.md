# Summary of Uncommitted Changes - 2026-05-02

## High-Level Summary
The current uncommitted changes encompass three major features and improvements to the application:
1. **Full Rebranding to NABDA:** The app has been completely rebranded from "HealthSync" to "NABDA". This includes changes to the app icons, platform manifests (Android, iOS, Web), documentation, localization strings, and the splash screen, which now features a dynamic "Heartbeat" (pulse) animation.
2. **Profile Picture & Image Cropping:** Added the ability for users to upload and crop profile pictures. A new `UserAvatar` widget was created and integrated across the app (dashboard, patient lists, chats) to ensure profile images are displayed consistently. The backend models and UI were updated to support Base64 image strings and `profileImageUrl`.
3. **Notification Settings Enforcement:** Notification services were updated to respect the user's notification preferences. Push notifications and in-app alerts will now only be displayed if `enableNotifications` is true in the user settings.

## Purpose of Each Change
* **Rebranding**: To launch the application under the new "NABDA" brand identity, ensuring a modern and consistent visual experience across all platforms.
* **Profile Images**: To personalize the user experience, allowing doctors and patients to recognize each other through custom profile pictures, similar to standard chat applications.
* **Notification Fixes**: To improve user privacy and control by strictly adhering to the user's notification preferences.

## Potential Risks & Things That Need Review
* **Image Upload Payload Size**: The profile pictures are encoded in Base64 before sending to the backend. Large images might cause payload size issues if the backend does not have a high enough limit configured. The `image_cropper` and `image_picker` compress the image, but it still warrants testing.
* **Push Notification Channel ID Change**: The channel ID in Android was updated to `nabda_alerts_v1`. Existing users on Android who update the app will have a new notification channel created for them.
* **Platform Configurations**: Changes to `AndroidManifest.xml` and `Info.plist` should be double-checked during the release build to ensure no bundle identifiers or critical metadata were unintentionally altered.
